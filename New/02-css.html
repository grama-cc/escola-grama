Css

O cascading Style Sheets (CSS) é uma "folha de estilo" utilizada para definir a aparência de uma página da internet. O CSS define como serão exibidos os elementos do html e é onde colocamos os estilos do site. O mais importante do css é separar a formatação do conteúdo de um documento. Ou seja:

  • html - Conteudo do site
  • css - Estilos do site

Podemos colocar o .css de 3 formas diferentes no nosso arquivo.

  1. Inline style - Estilo na linha do html
  2. Internal style sheet - Folha de estilo interna, dentro do .html
  3. External style sheet - Folha de estilo externa, dentro de um arquivo .css

Essa lista está organizada do mais especifico para o menos especifico. Portanto, um css Inline irá sobrescever um estilo de folha interna ou externa. Da mesma forma como um css escrito em folha interna irá sobrescrever um .css que esteja em folha externa.

Mas como que funciona? Ok, vou explicar

  1. Inline style - Estilo na linha do html

    Essa não é e melhor forma de se aplicar o css já que mistura estilo e conteudo. O ideal é manter conteúdo em .html e estilo em .css

  2. Internal style sheet - Folha de estilo interna

    Abrimos uma tag "style" na "head" e colocamos os estilos dentro dela, você pode ver isso aplicado na "head" deste arquivo.

  3. External style sheet - Folha de estilo externa

    Colocamos um "link" na "head" - link rel="stylesheet" type="text/css" href="css/style.css" - e colocamos os estilos em um outro arquivo .css que deve estar em uma pasta "css" junto com os arquivos "html". Você pode ver o link aplicado na "head" desse arquivo e observar a organização das pastas nos nossos arquivos.

Ids e Classes

Para aplicarmos o css precisamos selecionar partes do html. Podemos fazer isso usando os seletores, como "p", "ul", "li", "div", classes (class) ou Ids (identity). Não podemos ter mais de um Id em cada página. O ideal é manter as Classes e os Ids com nomes em inglês e que façam sentido na identificação posterior.

    Como funciona a Cascata?

Colocamos na "Head", dentro de "style" os estilos da Id e da Classe. A notação da Id é "#" enquanto a da Classe é "."

Se existirem informações conflitantes, por exemplo: "Cor das "ul" é roxo" e "cor da classe ".item" é Azul" a informação que vai valer é a que é mais especifica. Caso alguma informação se repita no .css a informação que foi escrita na linha de numero mais alto vai sobreescrever a escrita anteriormente

Temos também o terceiro item da nossa lista. Ele está em preto já que dissemos que todos os "p" dentro de "ul"s devem estar em preto. Por isso o ponto e o texto anterior continuam em roxo enquanto a segunda pergunta está em preto.

Mas por que ele você colocou "margin: 0;" no "ul p"?

Muitos dos seletores tem alguns estilos que vem independente de nós colocarmos. Um bom exemplo disso é o "p". Para que ele fique com a mesma distancia dos outros itens da "ul" devemos zerar a "margin" ou seja, a margem. Uma experiencia bacana é comentar ("command" + "/") a linha 19 do código, ou seja o "margin: 0;" dentro do "style" que está dentro do "head" e atualizar a página no seu browser para ver o que acontece.

Pseudo-classes

As peseudo-classes são classes que os nossos elementos recebem automaticamnete devido ao seu estado. Alguns exemplos de pseudo-classes são: visted (link visitado), hover (mouse em cima do link) e active (link selecionado, clicando nele)
